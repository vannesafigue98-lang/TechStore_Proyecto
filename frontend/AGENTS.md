# Instrucciones para Codex

Este es un proyecto educativo FullStack para estudiantes principiantes

## Estructura del proyecto

- `frontend/`: contiene las páginas HTML, CSS, imágenes, videos y JavaScript. - `backend/`: contiene el servidor Node.js con Express.

## Reglas generales 

1. No cambiar la estructura de carpetas.
2. No mover archivos sin autorización.
3. No eliminar código existente sin explicar el motivo.
4. Mantener el código sencillo, claro y comentado.
5. Usar buenas prácticas básicas de HTML, CSS, JavaScript y Node.js.
6. Explicar brevemente los cambios realizados.
7. No modificar archivos fuera del alcance solicitado.

## Reglas para frontend

Cuando la tarea sea de frontend:

 - Trabajar solo dentro de la carpeta `frontend/`. 
 - Usar Bootstrap 5 mediante CDN. 
 - Mantener las páginas: 
    - `index.html` 
    - `ayuda.html` 
    - `contacto.html` 
- Usar `css/estilos.css` para estilos personalizados. 
- Usar `js/script.js` para JavaScript propio. 
- No modificar la carpeta `backend/`.

## Reglas para el backend

Cuando la tarea sea de backend: 

- Trabajar solo dentro de la carpeta `backend/`. 
- Mantener el archivo principal como `server.js`. 
- Usar Node.js y Express. 
- No modificar la carpeta `frontend/`, salvo que la tarea indique integración. 
- Explicar cómo ejecutar el servidor.

## Reglas para integración

Cuando la tarea sea de integración: 
- Revisar tanto `frontend/` como `backend/`. 
- Usar `fetch` desde el frontend para comunicarse con el backend. 
- Verificar que las rutas del backend funcionen correctamente. 
- Mantener el código comprensible para estudiantes.
